# 2.1 Naming Rules

**"Naming Rules"** significa reglas de nombrado. En programación son las normas que determinan cómo puedes poner nombres a variables, funciones o cualquier identificador.

---

En JavaScript estas reglas indican qué es válido y qué no al momento de nombrar algo.

---

## Regla 1: Primer carácter válido

Un nombre en JavaScript debe empezar con una **letra**, con un **guion bajo (_)** o con un **signo dólar ($)**. Por ejemplo:
```javascript
let nombre;
let _usuario;
let $precio;
```

---

## Regla 2: No empezar con números

Un nombre **no puede empezar con un número**. Por ejemplo:
```javascript
let 1persona; // Esto da error.
```

---

## Regla 3: Números después del primer carácter

Después del primer carácter, el nombre **sí puede contener números**. Por ejemplo:
```javascript
let persona1 = "Ana";
```

---

## Regla 4: No usar espacios

Los nombres **no pueden tener espacios**. Por ejemplo:
```javascript
let nombre completo; // Esto da error.
```

---

## Regla 5: Sensibilidad a mayúsculas/minúsculas

Los nombres **distinguen entre mayúsculas y minúsculas**. Eso significa que estas dos variables son diferentes:
```javascript
let nombre;
let Nombre;
```

---

## Regla 6: Palabras reservadas

**Tampoco puedes usar palabras reservadas** del lenguaje como `class`, `return`, `let`, `function`, `if`, `while`, etc. Por ejemplo:
```javascript
let class = 5; // Esto da error.
```

> **Nota:** Las palabras reservadas del lenguaje son palabras que usa JS en otro tipo de cosas en el código, con lo cual no pueden ser nombradas porque se rompen o declaran otras cosas, por eso da error de manera automática.

---

## Buenas prácticas

Aunque no es obligatorio, es **recomendable usar nombres descriptivos** para que el código sea más claro. Por ejemplo:
```javascript
let edadUsuario; // Buen nombre.
let x; // Válido, pero mala práctica porque no describe nada.
```
